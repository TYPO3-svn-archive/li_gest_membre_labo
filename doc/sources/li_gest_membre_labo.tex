\chapter{Installation et Utilisation de l'extension de Gestion de Membres}

\section{Installation}

L'installation de l'extension de Gestion de Membres est la même que pour toute extension Typo3. Dans le menu de droite, on choisi "Admin Tool", puis "Gest d'Extensions" et soit "Install Extensions" (si l'extension est déjà présente en local), soit "Import Extensions" (pour installer l'extension à partir du fichier .t3x)\\

Une fois l'extension installer sur le serveur, il faut ensuite l'activer dans Typo3. Pour cela, rendez-vous dans le gestionnaire d'extensions dans la partie "Frontend Plugins". ici vous trouverez celle qui nous intéresse:\\

\begin{figure}[ht!]
	\centering
	\includegraphics[width=15cm]{images/Membres/Installation/installation1}
	\caption{Installation: Frontend Plugins}
	\label{fig:instal1}
\end{figure}

Cliquez sur le "$+$" devant l'extension pour lancer son installation. Une nouvelle page décrivant les tables à créer ou modifier s'ouvre alors.\\

\newpage

\begin{figure}[ht!]
	\centering
	\includegraphics[width=15cm]{images/Membres/Installation/installation2}
	\caption{Installation: Création des tables}
	\label{fig:instal2}
\end{figure}

Descendez jusqu'en bas de la page et cliquez sur "Make Update".\\

L'installation est maintenant terminée!\\

\begin{figure}[ht!]

Les champs des formulaires ont tous les mêmes caractéristiques:\\
\begin{itemize}
	\item Un champ contenant uniquement une boîte dialogue tel que le champ "Le nom de famille avant tout mariage" peut contenir tout type de caractères. Si ce champ est précédé d'un point d'exclamation comme pour le champ "Prénom", celui signifie qu'il est obligatoire.\\
	\item L'enregistrement ne sera donc pas accepté si le champ est vide. Les boîte de sélection comme celle du champ "Genre" s'utilise comme n'importe qu'elle boîte de sélection. S'il y a lieu, un champ vide est disponible.\\
	\item Pour les champs comportant des dates doivent être saisis sous la forme aaaa-mm-jj. Ainsi, le 11 décembre 1816 (Java est rendu aux Pays-Bas par les anglais...) devient 1816-12-11. Une date peut être obligatoire (un point d'exclamation est situé devant). Il est à noter que si vous sauvegarder sans avoir remplis une date obligatoire, se champs sera automatiquement remplit par la date du jour. Si vous laissé un champ date vide et que celui-ci n'était pas obligatoire, il sera automatiquement remplis par la date 0000-00-00. Une date 0000-00-00 concernant la fin de quelque chose sera interprétée par l'extension comme une date infinie (Si par exemple un membre est toujours au laboratoire mais on ne sait pas encore quand il va partir, on remplit ce champ par 0000-00-00).\\
\end{itemize}

\begin{figure}[ht!]

L'onglet filtres permet de sélectionner différentes catégories de membres. Il est ainsi possible de sélectionner les membres par rapport:\\

\begin{itemize}
	\item aux dates d'arrivée et de départ du membre dans le labo (Actuels, Anciens, Tous). Par defaut, "Actuels" est sélectionné.\\
	
	\item aux types de postes des membres. Si aucun poste n'est sélectionné, cela est considéré comme si tous les postes étaient sélectionnés. Il est possible d'ajouter une option sur la "période" à laquelle ce poste à été exercé (Actuels, Anciens, Tous). Par defaut, "Tous" est sélectionné.\\

	\item à la catégorie des membres. Si aucune catégorie n'est sélectionné, cela est considéré comme si toutes les catégories étaient sélectionnés. Il est possible d'ajouter une option sur la "période" à laquelle cette catégorie à été occupée (Actuels, Anciens, Tous). Par defaut, "Tous" est sélectionné.\\

	\item aux structures dans lesquelles travaille ou ont travaillé les membres. Si aucune structure n'est sélectionné, cela est considéré comme si toutes les structures étaient sélectionnés. Il est possible d'ajouté une option sur la "période" à laquelle le membre a appartenu à une des structures sélectionées (Actuels, Anciens, Tous). Par defaut, "Tous" est sélectionné.\\

	\item aux fonctions des membres. Si aucune fonction n'est sélectionné, cela est considéré comme si toutes les fonctions étaient sélectionnés. Il est possible d'ajouté une option sur la "période" à laquelle cette fonction a été occupée (Actuels, Anciens, Tous). Par defaut, "Tous" est sélectionné.\\

	\item aux diplômes des membres. Si aucun diplôme n'est sélectionné, cela est considéré comme si tous les diplômes étaient sélectionnés.\\

\end{itemize}

L'Onglet Requête permet de créer sa propre requête. Pour que la requête soit effectivement utilisée, il faut tout d'abord cocher la case "Utiliser une requête pour afficher les membres".\\

La construction de la requête se fait de la même manière que tout requête SQL, à l'exeption près, qu'elle est séparée en plusieurs parties. Ainsi vous devez écrire la partie SELECT de votre requête dans le champs SELECT, la partie FROM dans le champs FROM,... Les mots-clés SELECT, FROM, WHERE, ORDER BY,... ne doivent pas être indiqué.

Pour que cette requête fonctionne, certains informations sont obligatoires. Vous devez absolument prendre pour requête de départ celle qui est sur l'exemple ci-dessus, à savoir:\\

\begin{itemize}
	\item "DISTINCT tx\_ligestmembrelabo\_MembreDuLabo.uid AS uidmembre, tx\_ligestmembrelabo\_MembreDuLabo.*" dans le champ SELECT.\\
	\item "tx\_ligestmembrelabo\_MembreDuLabo" dans le champ FROM.\\
	\item "tx\_ligestmembrelabo\_MembreDuLabo.deleted<>1" dans le champ WHERE.
\end{itemize}

CEtte requête permet d'afficher tous les membres qui n'ont pas été effacé. Vous pouvez ensuite rajouter vos propres conditions...

\section{Création et utilisation de gabarits}

Dans Typo3, les gabarits (ou Templates) permettent de personnaliser l'affichage dans le Frontend à l'aide de balises et de marqueurs particuliers.\\

Dans cette extension, deux gabarits sont disponibles mais vous pouvez créer le votre pour obtenir un affichage personnalisé en fonction de vos besoins.\\

Lors de la configuration du plugin, pour indiquer le gabarit à utiliser, il faut remplir le champs de "Référence vers le fichier du gabarit HTML" de l'onglet "Gabarit".\\

\begin{figure}[ht]
	\centering
	\includegraphics[width=10cm]{images/Membres/Gabarit_Membres/Onglet}
	\caption{Onglet gabarit}
	\label{fig:onglet}
\end{figure}

Les deux gabarits disponibles sont:\\

\begin{itemize}
\item \verb+EXT:li_gest_membre_labo/pi1/li_gest_membre_labo_template.html+\\
\item \verb+EXT:li_gest_membre_labo/pi1/li_gest_membre_labo_template_libelleweb.html+\\
\end{itemize}

Le premier permet d'afficher la liste des membres (NOMs et Prénoms). Le deuxième affiche la liste des membres avec les NOMs et Prénoms ainsi ainsi que le Libellé Web leur correspondant. Les noms et prénoms renvoient vers le site internet du membre si celui-ci en possède un.

\subsection{Créer son gabarit}

Un gabarit est créé sur la base d'une page HTML (en-tête, corps,...). Dans le corps de cette page (body), on peut ensuite rajouter différentes balises pour obtenir un affichage personnalisé. Pour l'extension de Gestion des Membres certaines balises sont obligatoires, il s'agit des balises TEMPLATE (début du gabarit), CONTENT (début d'un contenu) et ITEM (liste des informations à afficher) qui permettre de structurer l'affichage.

Les balises s'utilisent toutes de la même façon:\\

\begin{itemize}
	\item une balise de départ: \verb+<!-- ###NOM### begin-->+\\
	\item une balise de fin:\verb+<!-- ###NOM### end-->+\\
\end{itemize} 

La structure de base d'un gabarit est:\\

\noindent \colorbox[gray]{0.9}{\begin{minipage}{0.9\textwidth}\parindent 15pt
\noindent $<$!-- \#\#\#TEMPLATE\#\#\# begin--$>$\\
\hspace*{1cm}$<$!-- \#\#\#CONTENT\#\#\# begin --$>$\\
\hspace*{1.5cm}$<$!-- \#\#\#ITEM\#\#\# begin --$>$\\
\hspace*{1.5cm}$<$!-- \#\#\#ITEM\#\#\# end --$>$\\
\hspace*{1cm}$<$!-- \#\#\#CONTENT\#\#\# end --$>$\\
\noindent $<$!-- \#\#\#TEMPLATE\#\#\# end--$>$
\end{minipage}}\\
~~\\

Nous pouvons maintenant ajouter dans ce code différentes balises, qui permettent un affichage de plusieurs éléments (quand la base de données renvoie plusieurs résultats) ou des marqueurs, qui n'affiche qu'un seul texte.

Une balise \verb+###PageWebLien###+ permet de créer un lien autour du texte que l'on veut. Ce texte peut être formé de marqueur ou non.\\

Par exemple pour afficher un lien autour du nom et du prénom d'un membre on écrit:\\

\noindent \colorbox[gray]{0.9}{\begin{minipage}{0.9\textwidth}\parindent 15pt
\noindent $<$!-- \#\#\#ITEM\#\#\# begin --$>$\\
\hspace*{1cm} $<$p$>$\\
\hspace*{1.5cm}$<$!--\#\#\#PageWebLien\#\#\# begin --$>$\#\#\#NOMDUSAGE\#\#\# \#\#\#Prenom\#\#\#$<$!--\#\#\#PageWebLien\#\#\# end --$>$\\
\hspace*{1cm} $<$/p$>$\\
\noindent $<$!-- \#\#\#ITEM\#\#\# end --$>$
\end{minipage}}\\
